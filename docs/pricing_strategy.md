## 一、定价哲学

```
┌──────────────────────────────────────────────┐
│  第三层: 价值定价                              │
│  ─ 独家模型 (Jais/Falcon/SEA-LION)            │
│  ─ 阿拉伯语优化不加价（战略性亏损获客）          │
│  ─ 企业 SLA + 数据主权 = 溢价 40%+             │
├──────────────────────────────────────────────┤
│  第二层: 竞争定价                              │
│  ─ OpenAI/Claude/Gemini 对标 OpenRouter 打 9 折 │
│  ─ 开源模型 (Llama/Mistral) 对标 Groq 价格      │
│  ─ 深度求索模型保持原价 (已是最低价)             │
├──────────────────────────────────────────────┤
│  第一层: 成本定价 (底价)                        │
│  ─ 上游 API 成本 × 1.08 (最低加价率)            │
│  ─ 覆盖带宽 + GPU 推理成本                     │
│  ─ 仅适用于大批量企业合同                       │
└──────────────────────────────────────────────┘
```

### 1.2 为什么不能单纯 "比 OpenRouter 便宜"？

| 策略 | 结果 |
|---|---|
| 全面降价 20% | 毛利转负，3 个月烧光 |
| 选择性降价 (区域模型) | 差异化的同时保持利润 |
| 独有模型高价 | 无竞争对手，定价权在自己手里 |
| 通用模型打 9 折 | 用户感知"更便宜"但不伤利润 |
| **结论** | **混合定价 = 引流模型薄利 + 独家模型高利** |

## 三、区域独家模型定价（核心竞争力）

### 3.1 中东区域模型

| 模型 | 能力 | 市场稀缺度 | **定价策略** | 预计毛利率 |
|---|---|---|---|---|
| **G42 Jais 30B** | 阿拉伯语生成、理解、翻译 | ⭐⭐⭐⭐⭐ 唯一性 | **$0.15/$0.15** (输入/输出同价) | 30-40% |
| **G42 Jais 70B** | 高级阿拉伯语推理 | ⭐⭐⭐⭐⭐ | **$0.35/$0.35** | 35-45% |
| **Falcon 180B** | 多语言 (含阿拉伯语) | ⭐⭐⭐ | **$1.00/$2.00** | 25-35% |
| **Falcon 40B** | 轻量级阿拉伯语 | ⭐⭐⭐⭐ | **$0.25/$0.50** | 30-40% |
| **Noor (沙特自研)** | 沙特方言特化 | ⭐⭐⭐⭐⭐ (如能接入) | **$0.20/$0.20** | 40-50% |

**为什么可以定高价？**

> OpenRouter 上没有 Jais。用户在 OpenRouter 上使用 GPT-4o 处理阿拉伯语 = 3x token 成本。
> 我们的 Jais 定价虽然 $0.15/token 看起来比 GPT-4o-mini 贵，但阿拉伯语场景下实际成本更低。

**实际案例：**

```
同一段阿拉伯语新闻 (500 词)：

GPT-4o 处理：
→ Tokenization: 500 词 ≈ 2,200 tokens (阿拉伯语膨胀)
→ 成本: 2,200 × $0.0025 = $5.50 (输入) + 输出约 $3.00 = $8.50

Jais 30B 处理：
→ Tokenization: 500 词 ≈ 600 tokens (原生阿拉伯语 tokenizer)
→ 成本: 600 × $0.00015 = $0.09 (输入) + 输出约 $0.15 = $0.24

成本差异: $8.50 vs $0.24 (35 倍！)
→ Jais 即便定价 $0.50/token 也远低于 GPT-4o 的阿拉伯语实际成本
```

### 5.2 本地货币动态定价

```
实时汇率引擎：
每日 UTC 00:00 抓取 EGP/IDR/VND/THB 对 USD 汇率
→ 自动调整本地价格展示
→ 后端实际结算: USDT (TRC-20) 或本地支付网关汇率

风险对冲：
埃及 EGP 波动大 → 每日调价 + 大额交易强制 USDT
印尼 IDR 相对稳定 → 月调价即可
```

## 八、竞争反制预案

### 如果 OpenRouter 跟进降价...

```
我们的应对:
1. 不全面跟进降价（那是自杀）
2. 强调"同样价格下的本地价值":
   - 阿拉伯语延迟 -90%
   - 本地支付（OpenRouter 没有 MADA/GrabPay）
   - 独家模型（OpenRouter 没有 Jais）
3. 锁定年合同用户（已签约的不会因为 5% 价差流失）
4. 加大免费额度（留存优先于收入）
```

### 如果本地出现竞品...

```
信号: 某沙特/印尼公司推出类似服务

我们的应对:
1. 独家模型协议加入排他条款
2. 加速支付能力建设（先发优势）
3. 降价打击（有融资支撑，本地竞品资金少）
4. 收购邀约（打不过就买）
```

### 如果上游供应商提价...

```
信号: OpenAI 提价 30%

我们的应对:
1. 合同预留调价条款
2. 自动转发生效
3. 同时主动引导用户转向开源替代
4. 毛利缓冲 (8-15% 加价率可吸收小幅提价)
```
